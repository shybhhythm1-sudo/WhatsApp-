# دليل تفعيل الإشعارات (Push Notifications) في تطبيق هيي

## الملفات اللي معاك دلوقتي
- `index.html` — الكود الرئيسي بعد التعديل (فيه طلب الإذن، تسجيل الـ Service Worker، حفظ التوكن، بانرات التثبيت والإشعارات)
- `firebase-messaging-sw.js` — الـ Service Worker اللي بيستقبل الإشعار وقت ما التطبيق يكون مقفول
- `manifest.json` — عشان زرار "أضف للشاشة الرئيسية" يظهر ويشتغل صح
- `functions/index.js` — الكود اللي بيشتغل على سيرفر Firebase ويبعت الإشعار الفعلي
- `functions/package.json` — المكتبات المطلوبة للـ Function
- `firebase.json` — إعدادات النشر

---

## الخطوة 1: احصل على VAPID Key

1. روح لـ [Firebase Console](https://console.firebase.google.com) → اختار مشروع `heye-ee886`
2. من القائمة الجانبية: **Project settings** (⚙️ بجانب Project Overview)
3. تاب **Cloud Messaging**
4. لو مش لاقي تاب Cloud Messaging، لازم تفعّل **Firebase Cloud Messaging API** الأول من [Google Cloud Console](https://console.cloud.google.com/apis/library/fcm.googleapis.com) (اختار نفس المشروع)
5. تحت قسم **Web configuration** → **Web Push certificates** → دوس **Generate key pair**
6. هيظهرلك مفتاح طويل (Key pair) — انسخه

7. افتح `index.html` ودور على السطر ده:
```js
window.__HEYE_VAPID_KEY__ = "PASTE_YOUR_VAPID_PUBLIC_KEY_HERE";
```
والصقه مكان النص:
```js
window.__HEYE_VAPID_KEY__ = "BXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX";
```

---

## الخطوة 2: فعّل خطة Blaze (Pay as you go)

الـ Cloud Functions محتاجة خطة Blaze (فيها free tier كبير، مش هتدفع غالبًا إلا لو عندك آلاف المستخدمين).

1. في Firebase Console → روح لـ **Usage and billing** (تحت ⚙️)
2. دوس **Modify plan** واختار **Blaze**
3. هيطلب منك تربط حساب فوترة (بطاقة ائتمان) — ده إجراء أمان من Google مش معناه هتتحصل فلوس فورًا

---

## الخطوة 3: ثبّت Firebase CLI على جهازك

افتح Terminal / Command Prompt على الكمبيوتر بتاعك (مش هنا في المحادثة) واكتب:

```bash
npm install -g firebase-tools
```

بعدين سجل دخول:
```bash
firebase login
```
هيفتحلك المتصفح تسجل بحساب Google بتاع المشروع.

---

## الخطوة 4: حضّر مجلد المشروع

1. حمّل كل الملفات اللي جهزتها لك (index.html, firebase-messaging-sw.js, manifest.json, firebase.json, functions/index.js, functions/package.json) في مجلد واحد على جهازك، مثلاً:
```
heye-app/
  index.html
  firebase-messaging-sw.js
  manifest.json
  firebase.json
  functions/
    index.js
    package.json
```

2. لازم كمان تضيف صورتين أيقونة للتطبيق في نفس المجلد بأسماء:
   - `icon-192.png` (192×192 بكسل)
   - `icon-512.png` (512×512 بكسل)
   (أي صورة مربعة للوجو بتاعك، PNG)

3. من جوه المجلد ده، شغّل:
```bash
firebase init
```
- اختار: **Hosting**, **Functions**, **Realtime Database** (بالمسافة Space تحدد، Enter تأكد)
- اختار مشروعك: `heye-ee886`
- Functions language: **JavaScript**
- استخدم ESLint؟ اختياري (ممكن "No")
- Install dependencies now? → **Yes**
- Hosting: public directory → اكتب `.` (نقطة، يعني المجلد الحالي)
- Single-page app? → **No**
- هيسألك يعمل overwrite لملفات موجودة (index.html, firebase.json) → قول **No** لأي ملف يسألك عليه عشان ميمسحش اللي جهزناه

---

## الخطوة 5: انشر كل حاجة

```bash
firebase deploy
```

ده هينشر الـ Hosting (الموقع) والـ Functions مع بعض. لو عايز تنشر حاجة واحدة بس:

```bash
firebase deploy --only hosting
firebase deploy --only functions
```

بعد النشر هيديك رابط زي:
```
https://heye-ee886.web.app
```

ده الرابط اللي تفتحه من الموبايل.

---

## الخطوة 6: جرّب الإشعارات

1. افتح الرابط من الموبايل (Chrome/Safari)
2. سجل حساب
3. هيظهرلك بانر "فعّل الإشعارات" — دوس تفعيل واقبل إذن المتصفح
4. من جهاز تاني (أو متصفح تاني) سجل حساب تاني وابعت رسالة للحساب الأول
5. لو التطبيق مقفول في الجهاز الأول، المفروض يوصله إشعار فعلي على الشاشة

---

## ملاحظات مهمة

- **iOS (آيفون):** الإشعارات في Safari مش بتشتغل إلا لو المستخدم عمل "Add to Home Screen" الأول وفتح التطبيق من الأيقونة (PWA mode)، وده متاح من iOS 16.4+. مينفعش الإشعارات تشتغل من جوه تاب Safari عادي في آيفون.
- **أندرويد:** بيشتغل عادي سواء من المتصفح أو بعد التثبيت.
- لو عايز تتأكد إن الـ Function شغالة، من Firebase Console → **Functions** → هتلاقي `sendMessageNotification` وتقدر تشوف الـ logs بتاعتها.
- الأيقونات (icon-192.png, icon-512.png) لازم تكونوا موجودين فعليًا وإلا الـ manifest والإشعارات هيبقوا بشعار افتراضي أو مكسور.

---

## لو حصل خطأ

- **"permission-denied" في الـ Function logs** → تأكد إن قواعد الـ Realtime Database بتسمح للـ Admin SDK (عادة بيكون افتراضيًا مسموح، الـ Admin SDK بيتخطى القواعد أصلاً)
- **مفيش إشعار وصل** → افتح Developer Tools في المتصفح → Console، شوف لو فيه خطأ يخص `getToken` أو `VAPID`
- **زرار "تثبيت" مش بيظهر** → ده بيظهر بس لو الموقع شغال على HTTPS (Firebase Hosting بيديك ده تلقائي) ولسه مش متثبت
